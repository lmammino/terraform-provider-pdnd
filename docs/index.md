# PDND Provider

The PDND provider enables Terraform to manage resources on the [PDND Interoperability](https://developers.interop.pagopa.it/) platform (API v3).

## Authentication

The provider uses DPoP (Demonstration of Proof-of-Possession) authentication. You need:

- An **access token** for the PDND API
- A **PEM-encoded private key** for DPoP proof generation
- A **key ID** identifying your DPoP key

## Example Usage

```terraform
# Auto-token mode (recommended): the provider obtains and refreshes tokens automatically.
provider "pdnd" {
  base_url         = "https://api.interop.pagopa.it/v3"
  client_id        = var.pdnd_client_id
  purpose_id       = var.pdnd_purpose_id
  dpop_private_key = file(var.dpop_private_key_path)
  dpop_key_id      = var.dpop_key_id
}

# Manual token mode (alternative): supply a pre-obtained access token.
```

## Schema

### Required

- `base_url` (String) Base URL of the PDND API (e.g., https://api.interop.pagopa.it/v3)
- `dpop_key_id` (String) Key ID for the DPoP private key
- `dpop_private_key` (String, Sensitive) PEM-encoded private key for DPoP proof generation

### Optional

- `access_token` (String, Sensitive) Access token for PDND API authentication (manual mode). Mutually exclusive with client_id/purpose_id.
- `client_id` (String) PDND client UUID for automatic token generation. Must be used together with purpose_id.
- `purpose_id` (String) PDND purpose UUID for automatic token generation. Must be used together with client_id.
- `request_timeout_s` (Number) Request timeout in seconds (default: 30)
- `token_endpoint` (String) PDND authorization server token endpoint (default: https://auth.interop.pagopa.it/token.oauth2)
